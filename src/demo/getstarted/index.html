Develop
Less/CSS

Introduction

样式部分由Less编写, 不建议CSS

目的样式的复用, 可读可维护

原则存在几何形状的组件以ui开头

尽可能嵌套进行复用, 命名尽量使用单个单词(从哲学角度讲, 命名越准确, 重用的可能性越小, 比如: .tab-nav-item和.item, 第二个命名就有可能用在tab或者menu等组件中, 所以样式名应尽可能抽象, 为后续重用提供可能)


Style Guide

请保持良好的代码结构, 保证代码的可读, 可维护

.box{border:1px solid #ddd;height:200px;width:200px;}
.box { border: 1px solid #ddd; height: 200px; width: 200px; .right { ... } .left { ... } }

4格缩进, 1个空格


References

参考资料 Semantic UI, Bootstrap

Javascript

Introduction

控件间应无耦合, 能组合存在或独立存在, 不应过分抽象, 保持简洁


Style Guide

请参考Google Javascript, Airbnb Javascript

请保持良好的排版和结构, 所有代码4格缩进

请使用Jshint做静态检查, 参考设置
